Interrupting commands

Commands can be interrupted, if the Go program supports it.

There are two ways of sending the interrupt signal to the program:

  1. If the program can handle INT signals, it indicates this by implementing the gogui_sigint command and returning its process ID. GoGui will then send an INT signal to the process if the user presses th interrupt button.
  2. If the program does not implement gogui_sigint, GoGui will send a special comment line (# interrupt) to the program. If the program reads the GTP input stream while processing a command, it can interrupt the command. Programs not aware of this convention will simply ignore the interrupt comment line.
